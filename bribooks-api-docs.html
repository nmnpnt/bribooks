BriBooks Platform
API Reference

Complete reference for the BriBooks book creation and publishing platform. All endpoints return JSON. Authenticated routes require a JWT Bearer token.

Version1.0.0

AuthJWT Bearer
Formatapplication/json

🔐
Authentication

BriBooks uses JWT (JSON Web Tokens) for authentication. After logging in, include the token in every protected request header.

Authorization Header Format
Authorization: Bearer <your_jwt_token>
Roles
author — creates and manages books  ·  reviewer — approves or rejects submitted books  ·  admin — publishes approved books
Token Expiry
Tokens expire after 60 minutes by default (configurable via JWT_TTL). Use POST /api/auth/refresh to renew before expiry.

Publishing Workflow

Books move through the following states. Each transition has strict role-based rules.

Draft
Author
→
Submitted
Author submits
→
Under Review
Reviewer
→
Approved
Reviewer approves
→
Published ✓
Admin publishes
Under Review
→
Rejected
Reviewer can reject from submitted or under review
Published books are permanently read-only. Any write operation on a published book will return 403 Forbidden.

Book Endpoints
GET /api/books List books ▼
Auth:All authenticated
Authors see only their own books. Reviewers and Admins see all books.
Query Parameters
ParamTypeRequiredDescription
statusstringoptionalFilter by: draft | submitted | under_review | approved | published | rejected
pageintegeroptionalPage number (default: 1, 15 per page)
// 200 OK — paginated { "data": [ { "id": 1, "title": "My Book", "status": "draft", /* ... */ } ], "meta": { "current_page": 1, "last_page": 3, "total": 42 } }
POST /api/books Create a new book ▼
Auth:Author only
Request Body (JSON)
FieldTypeRequiredDescription
titlestringrequiredBook title, max 255 chars
descriptionstringoptionalBook description / synopsis
genrestringoptionalGenre, max 100 chars
languagestringoptionalISO 639-1 code, 2 chars (default: en)
{ "title": "The Lost City", "description": "An epic adventure.", "genre": "Adventure" }
201Book created
403Non-author forbidden
422Validation failed
{ "message": "Book created successfully.", "data": { "id": 5, "title": "The Lost City", "status": "draft", /* ... */ } }
GET /api/books/{id} Get a single book ▼
Auth:All authenticated
Authors can only view their own books. Includes nested chapters and pages.
200Full book with chapters.pages and version count
403Author accessing another's book
404Book not found
PUT /api/books/{id} Update book metadata ▼
Auth:Author (owner only)
Cannot update published books. All fields are optional (partial update).
200Updated book data
403Not owner, or book is published
DELETE /api/books/{id} Soft-delete a book ▼
Auth:Author (owner only)
Soft delete — the record is preserved in the database with deleted_at set. Cannot delete published books.
200{ "message": "Book deleted successfully." }
403Not owner or book is published

Version Control

Each version stores a full JSON snapshot of the book's metadata, chapters, and pages. Versions are immutable and can be used to roll back the live book.

POST /api/books/{id}/versions Create a version snapshot ▼
Auth:Author (owner only)
Request Body (JSON)
FieldTypeRequiredDescription
labelstringoptionalHuman label e.g. "v1.0", "Final Draft"
change_notesstringoptionalDescription of what changed, max 1000 chars
// 201 Created { "data": { "id": 3, "book_id": 1, "version_number": 3, "label": "v1.2", "change_notes": "Revised chapter 2", "snapshot": { "metadata": { "title": "The Lost City", /* ... */ }, "chapters": [ { "id": 1, "title": "Chapter 1", "pages": [ /* ... */ ] } ] }, "created_at": "2024-01-15T12:00:00+00:00" } }
GET /api/books/{id}/versions List all versions ▼
Auth:All authenticated

Returns paginated list of versions ordered newest first (10 per page). Snapshot data is included in each item.

200Paginated version list
GET /api/books/{id}/versions/{versionId} Get a specific version ▼
Auth:All authenticated
200Full version with snapshot
404Version not found for this book
POST /api/books/{id}/versions/{versionId}/restore Rollback to a version ▼
Auth:Author (owner only)
Replaces all current chapters and pages with the snapshot data. The current live state is overwritten — create a new snapshot first if you want to preserve it.
200{ "message": "Book restored to version 2." }
403Not owner or book is published

Document Upload
POST /api/books/{id}/upload Upload manuscript (.doc / .docx) ▼
Auth:Author (owner only)
Async operation. The file is accepted immediately and conversion is dispatched as a background job. Returns 202 Accepted. The resulting chapters and pages appear once the job completes.
Request Body (multipart/form-data)
FieldTypeRequiredDescription
manuscriptfilerequired.doc or .docx file, max 20 MB
// 202 Accepted { "message": "Manuscript uploaded. Conversion is being processed in the background.", "file_path": "manuscripts/5/my-book.docx" }
403Not owner or book is published
422File missing, wrong format, or too large

Workflow Transitions
POST /api/books/{id}/submit Submit book for review ▼
Auth:Author (owner only)
Runs content moderation before submitting. If profanity or restricted words are found, the request is rejected with 422 and a list of violations. Valid from: draft or rejected status.
{ "message": "Book submitted for review.", "data": { "status": "submitted", "submitted_at": "2024-01-15T14:00:00+00:00", /* ... */ } }
{ "message": "Book failed content moderation and cannot be submitted.", "flagged_items": [ { "type": "profanity", "word": "****" }, { "type": "restricted_word", "word": "bomb" } ] }
POST /api/books/{id}/approve Approve a submitted book ▼
Auth:Reviewer only

No request body. Valid from: submitted or under_review. Notifies the author by email.

200{ "data": { "status": "approved", "approved_at": "..." } }
403Not a reviewer
422Book not in submittable state
POST /api/books/{id}/reject Reject a submitted book ▼
Auth:Reviewer only
Request Body (JSON)
FieldTypeRequiredDescription
reasonstringrequiredRejection reason, min 10 chars, max 1000 chars
{ "reason": "Content needs major revision in chapters 2 and 4." }
200Book status set to "rejected" with reason stored
422Reason field is required
POST /api/books/{id}/publish Publish an approved book ▼
Auth:Admin only
Once published, a book is permanently read-only. All write endpoints will return 403. Notifies the author by email.
200{ "data": { "status": "published", "published_at": "..." } }
403Not an admin
422Book not in "approved" state

Error Reference

All errors return JSON with at least a message field.

CodeMeaningCommon Cause
400Bad RequestMalformed JSON
401UnauthenticatedMissing or expired JWT token
403ForbiddenInsufficient role, not owner, or book is published
404Not FoundResource ID does not exist
422Validation ErrorMissing required fields or business rule violation
500Server ErrorDocument conversion failure, queue error
